1. ¿Qué datos tenemos? 📊

El dataset IRIS contiene información de 150 flores, de tres especies diferentes. Para cada flor, medimos:

  • 🌿 Largo y ancho del sépalo (la parte verde que protege la flor)
  • 🌸 Largo y ancho del pétalo (la parte colorida)

¿Cómo se ven los datos?

  • El largo del sépalo va de 4.3 a 7.9 cm (en promedio, 5.84 cm)
  • El ancho del sépalo va de 2.0 a 4.4 cm (en promedio, 3.06 cm)
  • El largo del pétalo va de 1.0 a 6.9 cm (en promedio, 3.76 cm)
  • El ancho del pétalo va de 0.1 a 2.5 cm (en promedio, 1.20 cm)

¿Y las especies?

Hay 50 flores de cada especie: setosa, versicolor y virginica. ¡Perfectamente balanceado!

2. ¿Qué relación hay entre las medidas? 🔗

A veces, cuando una flor tiene pétalos largos, también tiene pétalos anchos. Eso se llama "correlación". En nuestro caso:

  • El largo y ancho del pétalo están muy relacionados
  • El largo del sépalo también se relaciona bastante con el largo y ancho del pétalo
  • El ancho del sépalo es un poco más independiente

¿Qué significa esto?

Si conocemos el largo del pétalo, probablemente también sepamos si es ancho o no. ¡Las flores tienen sus secretos!

4. ¿Podemos agrupar o predecir la especie de una flor? 🤖

Probamos varios modelos de Machine Learning. Aquí te cuento cómo les fue, usando ejemplos sencillos:

a) K-means (Agrupamiento sin saber la especie)

  • Imagina que tienes que agrupar las flores solo mirando sus medidas, sin saber la especie
  • El modelo acertó el 44.00% de las veces cuando comparamos con las especies reales

¿Por qué 83.33% de accuracy?

El K-means agrupa las flores por similitud física, no por especie. Para calcular el accuracy:

  1. K-means encuentra 3 grupos basándose solo en las medidas (largo/ancho de sépalos y pétalos)
  2. Mapeamos cada cluster a la especie más frecuente en ese grupo
  3. Calculamos cuántas flores fueron clasificadas correctamente según este mapeo
Resultado del código base:
  • Cluster 1: 50 setosas (100% correctas) - Las setosas son muy fáciles de identificar
  • Cluster 2: 39 versicolor + 14 virginica (39/53 = 74% correctas)
  • Cluster 3: 11 versicolor + 36 virginica (36/47 = 77% correctas)

Total: 66 de 150 flores (44.00%) clasificadas correctamente según el código base

El mapeo manual de clusters a especies en el código base resulta en un accuracy menor que el mapeo automático basado en la especie más frecuente.

b) Modelos supervisados (¡Ahora sí sabemos la especie!)

Probamos 6 modelos diferentes. Cada uno aprende de forma única:

KNN 🌟

Busca las flores más parecidas y copia su especie

Ejemplo: "Esta flor se parece a 3 flores setosa cercanas → debe ser setosa"

SVM 📏

Dibuja líneas imaginarias para separar especies

Ejemplo: "Si pétalo > 2.5cm → virginica, si no → setosa"

Naive Bayes ⚡

Calcula probabilidades basado en características

Ejemplo: "90% de setosas tienen pétalos cortos → 90% probable que sea setosa"

Árbol de Decisión 🌳

Hace preguntas una por una hasta decidir

Ejemplo: "¿Pétalo largo? No → setosa. Sí → ¿Sépalo ancho? Sí → versicolor"

Random Forest 🌲

Varios árboles votan y gana la mayoría

Ejemplo: "3 árboles dicen setosa, 1 dice versicolor → setosa"

Red Neuronal 🧠

Aprende patrones complejos como el cerebro

Ejemplo: "Después de ver 100 flores, reconoce patrones sutiles"

7. ¿Qué aprendimos? 🤔

  • ¡Las flores del dataset IRIS son bastante fáciles de clasificar para los modelos de Machine Learning!
  • Si quieres la máxima precisión, usa una Red Neuronal
  • Si prefieres velocidad, Naive Bayes es tu amigo
